# SelfLoop SOM

자율 성장 의미지도(Growing SOM) 위에서 **스스로 학습하고, 그 지도를 가드레일 삼아 답하는** 실험용 시스템.
파인튜닝 없이 의미지도만 키웠을 때 (1) 지도가 고도화되는지 붕괴하는지, (2) 답변의 추론 궤적이 어떻게 변하는지를 관측한다.

> CPU-only · 경량 · 인간 개입은 "틀렸다" 신호 정도로 최소화하는 자율 루프 지향.

## 핵심 아이디어

- **SOM = 호기심**: 빈/고오차 영역을 찾아 그쪽을 더 공부할 곳으로 삼고, 노드를 늘려 스스로 자란다(GSOM).
- **Markov/BMU 거리 = 양심**: 새로 들어온 내용이 학습된 분포 안인지(도메인 내) 채점해 가드레일로 쓴다.
- **추론경로 = 논리의 발자국**: 질문 격자 → 답변 격자까지 의미 가중 최단경로의 길이를 잰다. 지도가 깊어지면 경로가 길어지고, 붕괴하면 한 점으로 직결된다.

## 페이지 1 — 학습 루프

- **자료 출처**: 코퍼스 직접 입력/업로드(.txt) **또는** "분야 검색해서 학습"(웹 크롤링)
- **라운드 옵션**: 라운드 수 / 1분 / 5분 / 10분
- **GSOM 자율 성장** on/off
- 실시간 계측(점유율·노드·평균 QE·어휘다양성)과 **붕괴 조기경보**
- 상태를 pkl로 저장

## 페이지 2 — 답변 루프

- pkl 불러오기(이전 학습 누적)
- 질문 → **SOM 가드레일**(BMU 거리로 도메인 이탈 판정) → 맥락 추출 → **LLM 호출**
- **추론경로** 표시: 의미 도정거리 / 거친 노드 수 / 경로


## 웹 검색(크롤링) — Brave Search API

자율 학습의 "스스로 검색" 기능은 **Brave Search API**로 동작한다(권장).

1. https://api-dashboard.search.brave.com 에서 무료 키 발급 (월 약 1000건 무료)
2. 앱 학습 루프 → "분야 검색해서 학습" → **Brave Search API Key** 입력
3. 검색어 입력 후 수집

- **본문까지 크롤링** 체크: 검색된 URL 본문을 직접 수집(알찬 코퍼스). 끄면 검색 스니펫만 사용(빠름).
- 키를 비우면 무료 검색엔진(DDG/Bing/Mojeek) 스크래핑으로 폴백하지만, 대부분 403으로 차단되므로 Brave 키 사용을 권장한다.
- 환경변수 `BRAVE_API_KEY`로도 입력 가능.

> 왜 API인가: ChatGPT 등 AI의 웹검색도 스크래핑이 아니라 검색 API를 쓴다. 검색엔진 결과 페이지 스크래핑은 구조적으로 차단되며(403), 헤더를 위장해도 오래가지 못한다.

## tok_emb 연결

사이드바에서 TinyTransformer `tok_emb` pkl을 올리면 실제 임베딩을 쓴다.
형식:

```python
{"word2idx": {단어: 인덱스, ...}, "tok_emb": np.ndarray(V, dim)}
```

업로드가 없으면 **결정론적 해시 임베딩**으로 폴백한다(네트워크 불필요, 파이프라인 검증용).

## pkl 누적 학습

SOM 가중치는 **INT8 양자화**로 저장(용량 4~7배 압축), 불러올 때 역양자화로 복원되어 같은 상태에서 학습을 이어간다.

## 관측된 결과 (합성 데이터 기준)

| 지표 | 고도화 | 붕괴 | 의미 |
|---|---|---|---|
| 점유율 | 유지 | 한 점 수렴 | 다양성 |
| 평균 QE | 안정 | 0으로 소멸 | 새 정보 유입 |
| 추론경로 비용 | 증가 | 고착 | 궤적 깊이 |
| 붕괴경보 | 안 뜸 | 발생 | 안전판 |

추론경로 고착이 점유율 붕괴경보보다 **먼저** 나타나, 붕괴 선행지표 후보가 된다.

## 알려진 제약 (정직한 메모)

- **크롤링·LLM**은 네트워크/API 키가 있어야 작동한다(로컬 환경 전용).
- **해시 폴백 임베딩**은 의미를 모르므로 맥락 추출 품질이 낮다 → `tok_emb` 투입 시 해결.
- **INT8 양자화**는 현재 노드 규모에서 용량 이득만 있고 속도 이득은 없다. 속도 이득은 노드 수천 이상 또는 정수 SIMD 직접 호출에서 나타난다.
- **추론경로 hop 절대값**은 노드 수에 비례하므로, 조건 비교 시 노드 수 정규화가 필요하다.
- 이 시스템이 잡는 것은 **도메인 이탈(out-of-domain)**이지 사실 오류(factual hallucination)가 아니다.

## 검색·크롤링 학습 기능

학습 루프의 `분야 검색해서 학습(크롤링)`에서 다음을 지원한다.

- DuckDuckGo HTML 검색으로 공개 웹페이지 URL 수집
- 검색 결과 페이지 수 조절
- 최대 수집 문장 수 조절
- 직접 URL 추가 크롤링
- 수집 소스 로그 확인
- 수집 코퍼스 `.txt` 다운로드
- 수집된 문장을 바로 학습 코퍼스에 추가

주의:

- 크롤링은 로컬 실행 환경에서 네트워크가 허용되어야 작동한다.
- 공개 웹페이지의 텍스트만 수집하는 용도로 설계했다. 로그인, 유료문서, 민감한 개인정보 페이지는 넣지 않는 것이 좋다.
- 크롤링 결과는 원문 품질 편차가 크므로, 중요한 도메인 학습에는 수집 코퍼스를 한 번 검토한 뒤 학습하는 것이 좋다.

## GPT / LLM API 호출 설정

`app_selfloop.py` 답변 루프에서 OpenAI 또는 OpenAI 호환 API를 직접 호출할 수 있다.

사이드바에서 설정:

- API Key: OpenAI API Key 또는 호환 서버 키
- Base URL: 기본값 `https://api.openai.com/v1`
- Model: `gpt-4o-mini`, `gpt-4.1-mini`, `gpt-4o`, 직접 입력 등
- System Prompt: 학습된 코퍼스 범위, 도메인 정체성 유지 지시문
- max_tokens: 응답 최대 토큰 수

로컬 LLM 서버를 OpenAI 호환으로 열었을 경우 예시:

```text
Base URL = http://localhost:1234/v1
Model = local-model-name
```

답변 루프에서 `질문·답변을 코퍼스에 저장`을 켜면, 생성된 Q·A가 코퍼스에 누적된다. 이후 학습 루프를 다시 실행하면 자기성찰 데이터가 SOM 의미지도에 반영된다.


## 크롤링 — PDF 자동 추출 & 깨진 텍스트 필터

- 크롤링 중 PDF 링크(content-type 또는 .pdf 또는 %PDF 매직넘버)를 만나면 자동으로 pdfplumber/pypdf로 본문 텍스트를 추출한다. (교육청·임용 자료는 PDF가 많아 필수)
- 수집 문장은 깨진 텍스트 필터를 통과해야 코퍼스에 들어간다: PDF 내부 구조(endobj/stream 등), 제어문자, 비정상 문자 비율, 의미 단어 부족 문장은 자동 제거.
- 이로써 "광고 보일러플레이트"(웹)와 "PDF 바이너리"(문서) 양쪽 잡음이 걸러진다.

## tok_emb 만들기 / 키우기 (씨앗 + 자율 성장)

해시 임베딩으론 SOM이 도메인을 못 가른다(분리지수 0.46). 실제 tok_emb를 쓰면 갈린다(2.35).

### 1) 처음 만들기 (코퍼스로 학습)
```bash
python build_tok_emb.py --pkl selfloop_state.pkl --out tok_emb.pkl --dim 64 --epochs 20
# 또는 txt에서
python build_tok_emb.py --txt corpus.txt --out tok_emb.pkl
```

### 2) 씨앗으로 키우기 (기존 임베딩 + 새 분야)
한국인이 모국어를 유지한 채 새 분야 전문어를 배우듯, 기존 tok_emb를 씨앗으로 두고 새 코퍼스로 성장시킨다. 기존 단어 의미는 보존되고(낮은 학습률 보호), 새 단어만 빠르게 학습된다.
```bash
python build_tok_emb.py --txt 새분야.txt --seed-emb tok_emb.pkl --out tok_emb.pkl --epochs 8
```

생성된 tok_emb.pkl을 앱 사이드바 "tok_emb pkl"에 올리면 적용된다.

### 엔진에서 직접 (GrowingEmbedding)
```python
from selfloop_engine import GrowingEmbedding
ge = GrowingEmbedding(dim=64)
ge.load_seed(tok_emb_path="tok_emb.pkl")   # 씨앗
ge.grow(new_sentences, epochs=5)            # 새 분야 자율 습득
ge.save("tok_emb.pkl")
```

## 마르코프 도메인 가드레일

학습된 코퍼스의 토큰 바이그램 전이확률(JM smoothing)로 새 문장을 채점(avg_logP)한다. 임계값보다 낮으면 "도메인 밖"으로 보고 코퍼스에 넣지 않는다 — 자율 수집 시 광고·잡음·이질 도메인을 거르는 문지기.

- 학습 루프에서 "🛡 마르코프 가드레일" 체크 시 적용
- 엄격도 슬라이더: 높일수록 많이 거부. 잡음 섞인 코퍼스는 25 이상 권장
- 거부된 문장은 "거부된 문장 보기"에서 검토 가능
- 코퍼스가 깨끗할수록 가드레일이 정밀해진다(garbage in이면 임계값이 물러짐)
- pkl 불러올 때 코퍼스로 가드레일 자동 재구성(옛 pkl도 호환)

## 자율 탐색 학습 (빈 격자 = 호기심)

학습 루프 → 자료 출처 → "🤖 자율 탐색" 선택 시:
1. GSOM의 저밀도 영역(덜 배운 곳)을 찾아 그 근처 단어로 검색어를 자동 생성
2. Brave로 크롤링
3. 마르코프 가드레일로 도메인 밖 문장 거부
4. 통과분만 학습

"SOM이 호기심, 마르코프가 양심" 구조. 인간 지정 없이 자란다.

- "🔮 자율 검색어 미리보기"로 어떤 검색어가 나올지 먼저 확인 가능
- 코퍼스의 도메인 균형이 탐색 방향을 결정한다(많이 아는 쪽을 더 깊이 판다)
- 검색어/가드레일 품질은 코퍼스 청결도에 비례

## 보상 기반 자율 학습 (교육학 + RL)

자율 수집에 "평가 → 보상 → 정책 조정" 루프를 더한 것. 정통 RL은 아니고 reward-guided curation.

**학습 보상 (learning_reward)** — 교육학의 근접발달영역(ZPD) 개념:
- 동화(assimilate): 새 자료로 QE가 내려가면 + (결국 녹아들었나)
- 난이도(difficulty): 적당히 도전하고 *결국 녹여낸* 경우만 보너스. 너무 쉬우면 작은 +, 너무 어렵거나(QE 폭증) 잡음이면 −
- 다양성(diversity): vocab 상승은 두 배 보상, 하락은 '모험'으로 절반만 벌점(작은 하락은 무시)

**검색어 정책 (QueryPolicy)** — 보상 높았던 키워드 방향을 강화, epsilon 확률로 새 방향 탐색(explore/exploit).

핵심: AI가 "잘 배워지는 주제는 더 깊이, 가끔 새 주제도 모험"하도록 자기 학습 방향을 보상으로 조절한다. = "AI 교육학"의 형성평가-피드백-교수조정 루프.

> 보상 설계 주의: QE만 보상하면 쉬운 자료만 모으는 함정(쉬운 것만 가르치는 수업) → 난이도·다양성을 섞어 방지.
